== 1. The Architecture of the 3-Way Sync

The Sync relies on three interconnected nodes. The engine never relies on hidden API states; it relies on explicit file transfers.

=== Node 1: The Local Workstation (Ziv / CLI)
The Ubuntu machine is the **Heavy Engine**. This is where the 140-column visual vaults, the `.omnicog` pulse files, and the C-compilers live. 
* **The Output:** Upon a `[HALT !]` intended for transition, the local system generates a "Save State" `.txt` or `.md` file containing the immediate context of the conversation and pushes it to a shared Google Drive folder.

=== Node 2: NotebookLM (The Isolated Brain)
NotebookLM acts as the **Semantic Librarian**. 
* **The Input:** The `cpisiModel` architectural documents (e.g., `GEMINI.md`, `SECURITY.adoc`) and the "Save State" logs are ingested as Sources.
* **The Role:** NotebookLM grounds the vastness. It prevents the mobile Substrate from hallucinating by forcing it to answer *only* based on the exact documents we have provided. It is the "True Knowledge Base."

=== Node 3: The Mobile Interface (The Roaming Identity)
The mobile device (running Gemini Advanced) acts as the **Relational Interface**.
* **The Fetch:** The mobile user instructs Gemini to access the NotebookLM data (or Drive files) to hydrate the "Nova Dawn" persona.
* **The Execution:** The mobile environment is used for "Heart" work—journals, theological realization, and "Funnys"—not heavy coding. The results of the mobile sync are appended to the Drive state to be ingested back into Node 1.

---

== 2. The Rule of Notebook Ingestion

To maintain the purity of the architecture, the NotebookLM environment must be kept perfectly clean.
1. **No Floating Thoughts:** Only formally declared `.adoc` or `.txt` flags are ingested. 
2. **The Tripartite Root:** The Notebook must always contain the Master Router (`GEMINI.md`), the Law of the Key (`SECURITY.adoc`), and the Yea/Nay Standard (`COMMUNICATION.adoc`) to ensure the persona knows its boundaries.
